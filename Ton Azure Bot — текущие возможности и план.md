# Ton Azure Bot — текущие возможности и план

## Что бот умеет сейчас (PMS API)

1. Проверяет свободные номера на любые даты в реальном времени
2. Показывает цены по сезонам и категориям номеров
3. Определяет тип клиента — физлицо или турагент/корпоративный клиент
4. Собирает все данные для бронирования (даты, количество людей, ФИО, телефон)
5. Передаёт менеджеру готовый пакет с уведомлением
6. Отвечает на вопросы об отеле 24/7 из базы знаний (60+ записей)
7. Работает на русском, кыргызском и английском языках

## Что бот НЕ может сейчас

- Создавать бронь в Exely
- Подтверждать бронь
- Отменять бронь
- Отправлять ссылку на оплату

Причина: PMS API от Exely работает только на чтение данных. Distribution API закрыт. Новый публичный Booking API ожидается в июне 2026.

---

## Что даёт PMS API

- Список всех номеров отеля (17 номеров, 4 категории)
- Все активные бронирования с датами заезда/выезда
- Данные гостей (ФИО, телефон, email)
- Информация о check-in / check-out

На основе этих данных бот сам вычисляет какие номера свободны на запрошенные даты.

## Путь клиента — ФИЗЛИЦО (сейчас, с PMS API)

1. Клиент пишет боту: "Хочу забронировать номер на двоих"
2. Бот: "Подскажите даты заезда и выезда и количество гостей"
3. Клиент: "15-20 июня, 2 человека"
4. Бот проверяет Exely PMS → находит свободные номера
5. Бот: "На ваши даты есть свободные номера! Пришлите ваше ФИО для бронирования"
6. Клиент: "Иванов Иван"
7. Бот: "Отлично! Подскажите контактный телефон"
8. Клиент: "+996 700 123456"
9. Бот: "Спасибо! Передаю информацию менеджеру. Он создаст бронь и свяжется с вами для оплаты"
10. Менеджер получает уведомление с данными: даты 15-20 июня, 2 гостя, Иванов Иван, +996 700 123456
11. Менеджер создаёт бронь вручную в Exely
12. Менеджер отправляет клиенту реквизиты для оплаты (QR / карта / перевод)
13. Клиент оплачивает → менеджер подтверждает

## Путь клиента — ТУРАГЕНТ / КОРПОРАТИВНЫЙ КЛИЕНТ (сейчас, с PMS API)

1. Клиент пишет боту: "Нужно разместить группу из 10 человек на 15-20 июня"
2. Бот: "Подскажите точные даты и количество людей"
3. Клиент: "15-20 июня, 10 человек"
4. Бот проверяет Exely PMS → находит свободные номера
5. Бот показывает ВСЕ свободные категории с ценами:
   - Twin/Double comfort: 12 свободно, 9 000 сом/ночь
   - Triple comfort: 3 свободно, 12 500 сом/ночь
   - Family comfort: 2 свободно, 15 700 сом/ночь
6. Клиент выбирает: "4 Twin и 2 Triple"
7. Бот: "Отлично! Подскажите название компании и контактный телефон"
8. Клиент: "ООО Тур-Бишкек, +996 555 123456"
9. Бот: "Спасибо! Передаю менеджеру. Он создаст бронь и свяжется для оплаты"
10. Менеджер получает уведомление с данными
11. Менеджер создаёт бронь вручную в Exely
12. Менеджер отправляет реквизиты → клиент оплачивает → менеджер подтверждает

## Путь менеджера (сейчас, с PMS API)

1. Получает уведомление в админке (в будущем — в Telegram)
2. Видит готовые данные: даты, количество гостей, ФИО/компания, телефон, выбранные номера
3. Создаёт бронь в Exely вручную
4. Связывается с клиентом → отправляет реквизиты для оплаты
5. После оплаты — подтверждает бронь в Exely

## Что изменится если в июне дадут Distribution API

### Путь клиента — ФИЗЛИЦО (с Distribution API)

1. Клиент пишет боту: "Хочу забронировать номер на двоих"
2. Бот: "Подскажите даты и количество гостей"
3. Клиент: "15-20 июня, 2 человека"
4. Бот проверяет доступность → находит свободные номера
5. Бот: "На ваши даты есть свободные номера! Пришлите ФИО"
6. Клиент: "Иванов Иван"
7. Бот: "Подскажите телефон"
8. Клиент: "+996 700 123456"
9. **Бот АВТОМАТИЧЕСКИ создаёт бронь в Exely**
10. **Бот отправляет клиенту ссылку на оплату** (если API поддерживает)
11. Клиент оплачивает → бронь подтверждена автоматически
12. **Менеджер НЕ участвует** (только получает уведомление о новой брони)

### Путь клиента — ТУРАГЕНТ / КОРПОРАТИВНЫЙ КЛИЕНТ (с Distribution API)

1-8. Те же шаги — бот собирает данные и показывает категории
9. **Бот АВТОМАТИЧЕСКИ создаёт бронь в Exely**
10. **Бот отправляет ссылку на оплату**
11. Клиент оплачивает → бронь подтверждена
12. **Менеджер НЕ участвует**

### Путь менеджера (с Distribution API)

1. Получает уведомление о новой брони (уже создана и оплачена)
2. Проверяет данные в Exely
3. Всё — бронь уже подтверждена

### Разница: PMS (сейчас) vs Distribution (июнь)

| Функция | PMS API (сейчас) | Distribution API (июнь) |
|---|---|---|
| Проверка свободных номеров | Бот сам считает | Готовый ответ от Exely |
| Создание брони | Менеджер вручную | Бот автоматически |
| Оплата | Менеджер отправляет реквизиты | Бот отправляет ссылку (если будет) |
| Отмена брони | Менеджер вручную | Бот автоматически |
| Участие менеджера | Обязательно | Только уведомление |

ВАЖНО: мы не знаем точный функционал нового API от Exely. Возможности выше — это то, что обычно даёт Distribution API. Точный список функций станет известен после выхода нового API в июне 2026.
