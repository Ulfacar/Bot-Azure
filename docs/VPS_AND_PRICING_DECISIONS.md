**Дата:** 08.04.2026

## 1. VPS для платформы

**Выбор: Contabo Cloud VPS 30 — $12/мес**
- 8 vCPU, 24 GB RAM
- Выделенный ТОЛЬКО под ASystem (никаких других проектов)
- Multi-tenant: один FastAPI + одна PostgreSQL на все отели
- Хватит на 10-30 клиентов

**Почему не дешевле:** текущий VPS (8 GB RAM) уже впритык с одним ботом + docker + другие проекты. 8 GB снова будет мало.

**Когда масштабировать на VPS 40 ($20.80):** при 30+ отелях или при добавлении тяжёлых фич (аналитика, real-time дашборды).

## 2. Ценообразование

**Модель: $700 разово + $20/мес подписка. Всё включено.**

- Один тариф для старта (без разделения по размеру отеля)
- $20/мес (~1600 сом) — психологически комфортно для КР рынка
- AI расход заложен в подписку, клиент не знает про токены
- Тарифную сетку вводим позже, когда будет 20+ клиентов и видна реальная разница в расходах

**Почему не $5/мес:**
- Не окупает даже VPS + AI расход
- Отели платят $700 за бота — бюджет на IT есть
- Конкуренты берут $50-200/мес
- При 10 клиентах по $5 = $50/мес (убыток), по $20 = $200/мес (прибыль)

## 3. OpenRouter / AI расход

**Решение: один общий аккаунт OpenRouter + трекинг по hotel_id**

Клиент НЕ создаёт аккаунт OpenRouter. Для него "бот просто работает".

**Реальный расход AI (Claude 3.5 Haiku):**
- Отель 17 номеров (~200 диалогов/мес): $0.50-1.00/мес
- Отель 50 номеров (~600 диалогов/мес): $1.50-3.00/мес
- 10 отелей суммарно: ~$10-20/мес

**Новая таблица в БД:**
```
ai_usage
├── id
├── hotel_id (FK → hotels)
├── conversation_id (FK → conversations)  
├── prompt_tokens
├── completion_tokens
├── model
├── created_at
```

При каждом вызове OpenRouter сохраняем usage из ответа.

## 9. Анализ конкурента: hotel-bot.ru

**Что у них:** "Премиальный AI-бот для гостиницы" — 24/7, Telegram, WhatsApp, виджет на сайт, 3 языка (RU/EN/中文), интеграция с PMS, white-label, передача менеджеру. Цены скрыты — "получить КП".

### Сравнение

| Фича | hotel-bot.ru | ASystem (мы) |
|------|-------------|--------------|
| AI бот 24/7 | Есть | Есть |
| Telegram + WhatsApp | Есть | Есть |
| Виджет на сайт | Есть | Нет (фаза 2) |
| Передача менеджеру | Есть | Есть |
| Языки | RU / EN / 中文 | RU / KY / EN |
| White-label | Есть | Есть (бот от имени отеля) |
| Интеграция PMS | "Под заказ" | Exely (работает), расширяемо |
| Цены | Скрыты (КП) | $700 + $20/мес — открыто |
| **Мини-чат превью** | Нет | **Есть** — клиент тестирует бота до покупки |
| **Self-service форма** | Нет (ручная настройка) | **Есть** — 5 шагов, клиент сам заполняет |
| **База знаний + обучение** | Не заявлено | **Есть** — менеджер обучает бота из чата |
| **Заметки менеджера** | Не заявлено | **Есть** — контекст звонков/визитов |
| **Реальные кейсы** | Нет на сайте | **86% эффективность, 90+ диалогов** |
| **Скорость подключения** | Неизвестно | **Быстро** — форма + активация |

### Наши преимущества

1. **Скорость** — главное преимущество. Клиент заполнил форму, протестировал бота, отправил заявку. Не ждёт КП неделю.
2. **Прозрачная цена** — $700 + $20/мес, без торга и КП. Клиент сразу знает что платит.
3. **Self-service** — минимум ручной работы с нашей стороны = масштабируемость.
4. **Проверено на реальном отеле** — Ton Azure, 90+ диалогов, 86% бот справился сам.

### Стратегия: скорость > код

Код дешевеет из-за AI. Через год-два конкурентов будет в разы больше. Задача сейчас — собрать клиентскую базу как можно быстрее, пока рынок КР/ЦА не насыщен. $700 за бота — правильная цена для скорости захвата рынка.

### Что взять у них на заметку (не в MVP, позже)

1. **Виджет на сайт** — клиенты будут спрашивать, добавить в фазу 2
2. **Китайский язык** — для крупных отелей с туристами из Китая (Иссык-Куль актуально)
3. **"Премиальное" позиционирование** — не стесняться называть продукт премиальным на лендинге

## 10. Путь клиента (Customer Journey)

### Сейчас (пока OS в разработке)

1. Продажница приезжает к отелю с презентацией
2. Демонстрирует живого бота Ton Azure — реальный кейс, 86% эффективность
3. Клиент говорит "Беру" → договор + оплата $700
4. Продажница заполняет бриф (BRIEF.md) с менеджером отеля
5. Alan получает бриф → вручную настраивает бота (1-2 дня)
6. Клиент получает логин/пароль в админку → бот работает

### В будущем (с OS платформой)

1. Продажница приезжает к отелю с планшетом
2. Открывает asystem.kg → "Создать бота для вашего отеля"
3. Заполняет форму 5 шагов вместе с клиентом (номера, цены, правила)
4. Шаг 5 — мини-чат превью: клиент видит как бот отвечает с данными ЕГО отеля (без ключей TG/WA, через общий OpenRouter)
5. Клиент говорит "Беру" → "Отправить заявку" → заявка в OS Dashboard (pending)
6. Договор + оплата $700
7. Alan в OS Dashboard: вставляет ключи (TG токен, wappi.pro, PMS) + "Активировать" → 30 минут вместо 2 дней
8. Клиент получает логин/пароль → заходит в свою админку → бот работает

### Что видит клиент (менеджер отеля) в админке

- Список диалогов с фильтрами (статус, категория, поиск)
- Чат с клиентом (может перехватить от бота и ответить сам)
- Шаблоны быстрых ответов (реквизиты, подтверждение, отказ, маршрут)
- База знаний (просмотр, добавление, редактирование)
- Кнопка "Обучить" — превратить диалог в запись базы знаний
- Заметки менеджера (привязаны к телефону клиента)
- Статистика (диалоги, эффективность бота)

## 11. План разработки

**Два параллельных трека:**

- **Трек 1 (деньги сейчас):** продажница ходит по отелям → демо Ton Azure → бриф → Alan настраивает вручную → доход идёт
- **Трек 2 (платформа):** Alan — multi-tenant backend, Emir — фронт OS → ускорение подключения с 2 дней до 30 минут

| Неделя | Alan (backend) | Emir (frontend) | Продажница |
|--------|---------------|-----------------|------------|
| 1 | Multi-tenant + prompt generator + webhook | Связывает backend с фронтом | Ходит по отелям, собирает брифы |
| 2 | OS Dashboard API (отели + активация) | OS Dashboard UI (отели + ключи) | Заключает первые договоры |
| 3 | Интеграция + тест на втором отеле | Лендинг + форма 5 шагов | Показывает работающего бота |
| 4 | Preview chat API + доработки | Мини-чат превью + заявки | Продаёт с демо на месте |

**Ключевой момент:** не ждём платформу чтобы продавать. Платформа ускоряет процесс, но продажи идут уже сейчас.

## 12. Улучшения для OS платформы (дополнения к ТЗ)

### Фаза 1 (добавить в MVP)

**1. Биллинг-трекинг**
При 10 клиентах потеряемся кто заплатил $20/мес, а кто нет. Минимум:
```
billing
├── id
├── hotel_id (FK → hotels)
├── month (2026-05)
├── amount
├── status: paid | pending | overdue
├── paid_at
```

В OS Dashboard — страница/блок: кто заплатил, кто должен, просрочка.

**Emir:** страница "Биллинг" в OS Dashboard — таблица оплат по месяцам, фильтр по статусу.
**Alan:** API `/api/admin/billing/`, таблица + миграция.

**2. Алерты в Telegram (для Alan)**
- Бот не отвечает > 5 минут → алерт
- Баланс OpenRouter < $20 → алерт
- Отель неактивен > 7 дней → алерт (клиент ушёл?)
- Новая заявка с сайта → алерт

**Alan:** фоновая задача проверки + отправка в TG через существующий механизм уведомлений.

**3. Мобильная адаптация админки отеля**
Менеджеры отелей НЕ сидят за компом — бегают по отелю с телефоном. Критично чтобы работало с мобильного:
- Список диалогов
- Чат с клиентом + перехват от бота
- Уведомления

**Emir:** сразу закладывать responsive дизайн в админке отеля. Не десктоп-first, а mobile-first. Это напрямую влияет на adoption — если менеджер не может ответить с телефона, он не будет пользоваться.

### Фаза 2 (после запуска, по мере роста)

**4. Обновление цен клиентом**
Сезон поменялся → отель хочет обновить цены. Сейчас: звонит Alan'у. Нужно: в админке отеля кнопка "Обновить информацию" → мини-форма (только цены и сезоны) → промпт перегенерируется автоматически.

**Emir:** форма обновления цен в админке отеля.
**Alan:** API перегенерации промпта из обновлённых данных.

**5. Онбординг менеджера**
При первом логине менеджера — короткий туториал (3-4 экрана): "Вот ваши диалоги", "Так перехватить чат", "Так обучить бота". Снижает нагрузку на поддержку.

**Emir:** 3-4 экрана-подсказки при первом входе (можно библиотека типа react-joyride).

**6. Виджет на сайт отеля**
Клиенты будут спрашивать. Встраиваемый чат-виджет (`<script>` тег) для сайта отеля.

**Emir:** виджет как отдельный JS bundle.
**Alan:** API для виджета (анонимный канал, без TG/WA).

**7. Реферальная программа**
Отельеры общаются друг с другом. Если клиент приводит другой отель → скидка месяц подписки. Бесплатный маркетинг.

**8. Годовая подписка**
Иссык-Куль = сезонный бизнес. Зимой бот почти не нужен, клиент может уйти. Решение: годовая подписка $200/год (вместо $240) — клиент платит вперёд, не уходит зимой, мы получаем кэш сразу.

**9. Кейс-страница на лендинге**
Блок с реальными цифрами: "Отель Ton Azure — бот обработал 90+ обращений, 86% без менеджера". С разрешения клиента. Это сильнейший продающий аргумент.

**Emir:** блок на лендинге с цифрами и отзывом.

**10. Китайский язык**
Для крупных отелей с туристами из Китая (Иссык-Куль актуально). Добавить 中文 как опцию при создании бота.
