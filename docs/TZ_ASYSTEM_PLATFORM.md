**Дата:** 07.04.2026
**Версия:** 1.0
**Команда:** Alan (ядро бота, backend), Emir (OS платформа, frontend)

---

## 1. Суть проекта

Asystem — multi-tenant SaaS-платформа для создания и управления AI-ботами отелей. Клиент (отель) заполняет форму на сайте, тестирует бота в мини-чате, отправляет заявку. Разработчик добавляет API-ключи и активирует бота. Менеджер отеля получает доступ к админке с диалогами.

## 2. Три интерфейса

### 2.1. Публичный сайт (клиент — владелец отеля)

**Цель:** клиент сам заполняет данные об отеле и тестирует бота.

**Лендинг:**
- Что такое Asystem, как работает
- Кейс: Тон Азур (реальные цифры — 2+ недели, 80+ диалогов)
- Цены: $700 подключение + $20/мес
- Кнопка "Создать бота для вашего отеля"

**Форма-шаблон (5 шагов):**

**Шаг 1 — "Расскажите о вашем отеле"**
- Название отеля (обязательно)
- Город / село / локация (обязательно)
- Краткое описание (2-3 предложения)
- Контактный телефон менеджера (обязательно)
- Имя менеджера (обязательно)
- Email
- Ссылка на Google Maps
- Сайт (если есть)

**Шаг 2 — "Какие номера у вас есть?"**
- Динамическая форма: "+ Добавить категорию номера"
- Для каждой категории:
  - Название (Twin/Double, Семейный и т.д.)
  - Количество номеров
  - Макс. гостей
  - Описание (кровати, вид, удобства)
- Сезоны: "+ Добавить сезон"
  - Название сезона
  - Даты (с — по)
- Таблица цен: сезон × категория × кол-во гостей (за сутки)
- Завтрак включён? (да/нет)
- Стоимость доп. питания
- Дети до ___ лет бесплатно

**Шаг 3 — "Правила для гостей"**
- Время заезда
- Время выезда
- Домашние животные (да/нет)
- Курение (да/нет/только на балконе)
- Предоплата для бронирования (%)
- Политика отмены (текст)
- Способы оплаты (чекбоксы: перевод, QR, карта, наличные)
- Реквизиты для перевода (текст)

**Шаг 4 — "Что ещё есть в отеле?"**
- Чекбоксы с полями:
  - [ ] Бассейн
  - [ ] Ресторан (вместимость: ___)
  - [ ] Конференц-зал (вместимость: ___, цена: ___)
  - [ ] Сауна / баня
  - [ ] Парковка
  - [ ] Wi-Fi
  - [ ] Трансфер (стоимость: ___)
  - [ ] Детская площадка
  - [ ] Пляж
  - [ ] Другое: ___
- Как добраться (расстояние от столицы, маршрут)
- Тон общения бота (дружелюбный / деловой / нейтральный)
- Языки (кыргызский, русский, английский — мультиселект)
- Что НЕ должен говорить бот (текст)
- Особые акции / скидки (текст)

**Шаг 5 — "Попробуйте вашего бота!"**
- Мини-чат превью прямо на странице
- System prompt генерируется автоматически из данных шагов 1-4
- Клиент пишет тестовые вопросы — бот отвечает в реальном времени
- Может вернуться на предыдущие шаги, исправить данные, попробовать снова
- Кнопка "Отправить заявку" → данные сохраняются в БД со статусом `pending`

### 2.2. OS Dashboard (разработчик — Asystem)

**Цель:** управление заявками, настройка ботов, мониторинг.

**Авторизация:** отдельная от клиентской, роль `superadmin`

**Страницы:**

**Заявки:**
- Список заявок (статус: pending → configuring → active → paused)
- Для каждой заявки:
  - Просмотр заполненного шаблона (все данные формы)
  - Редактирование данных (если клиент ошибся)
  - Форма технических ключей:
    - Telegram Bot Token
    - WhatsApp: Wappi API Key + Profile ID
    - PMS: тип (Exely/Bnovo/TravelLine/нет) + API Key + Hotel Code
    - OpenRouter API Key (или использовать общий)
  - Кнопка "Активировать бота" → создаёт запись в `hotels`, генерирует финальный промпт, запускает каналы
  - Кнопка "Приостановить бота"

**Активные отели:**
- Список всех активных ботов
- Для каждого:
  - Статус (online/offline)
  - Кол-во диалогов (всего / сегодня / активных)
  - Последняя активность
  - Быстрые действия: перейти в админку отеля, изменить конфиг, перезапустить, просмотреть логи
  - Расход AI-токенов

**Статистика (общая):**
- Всего отелей / активных / на паузе
- Всего диалогов за период
- Расход AI по всем отелям
- Доход (кол-во × подписка)

---

### 2.3. Админка отеля (менеджер отеля)

**Цель:** менеджер видит диалоги, перехватывает чат, управляет базой знаний.

**Основа:** текущая React-админка Тон Азура, адаптированная под multi-tenant.

**При логине:** менеджер видит ТОЛЬКО данные своего отеля (фильтр по `hotel_id`).

**Функционал (уже реализован в Тон Азуре):**
- Список диалогов с фильтрами (статус, категория, поиск)
- Чат с клиентом (перехват от бота, ответ вручную)
- Шаблоны быстрых ответов (реквизиты, подтверждение, отказ, маршрут)
- База знаний (просмотр, добавление, редактирование)
- Кнопка "Обучить" из диалога (вопрос → ответ в KB)
- Заметки менеджера (привязаны к телефону клиента)
- Статистика (диалоги, категории, активность)

### Multi-tenant подход
- Одна БД, одна инстанция FastAPI
- Все таблицы имеют `hotel_id`
- Row-level фильтрация при каждом запросе
- Один процесс обслуживает все отели

## 5. API-контракт

### Публичный сайт → Backend

```
POST /api/applications/
  Body: { form_data: {...}, contact_name, contact_phone, contact_email, hotel_name }
  Response: { id, status: "pending" }

POST /api/preview-chat/
  Body: { form_data: {...}, message: "Сколько стоит номер?" }
  Response: { reply: "Twin/Double — 8 000 сом/сутки..." }
  Примечание: НЕ сохраняет в БД, просто вызов OpenRouter
              с динамически сгенерированным промптом из form_data
```

### OS Dashboard → Backend

```
GET    /api/admin/applications/          — список заявок
GET    /api/admin/applications/{id}      — детали заявки
PATCH  /api/admin/applications/{id}      — обновить (статус, данные)
POST   /api/admin/applications/{id}/activate — активировать бота
  Body: { telegram_token, whatsapp_key, whatsapp_profile, pms_type, pms_key, pms_code }

GET    /api/admin/hotels/                — список активных отелей
GET    /api/admin/hotels/{id}/stats      — статистика отеля
PATCH  /api/admin/hotels/{id}            — обновить конфиг
POST   /api/admin/hotels/{id}/pause      — приостановить
POST   /api/admin/hotels/{id}/resume     — возобновить

GET    /api/admin/stats/                 — общая статистика платформы
```

### Админка отеля → Backend (уже существует, добавить hotel_id фильтр)

```
POST   /api/auth/login                   — логин (возвращает JWT с hotel_id)
GET    /api/conversations/               — диалоги (фильтр по hotel_id из JWT)
GET    /api/conversations/{id}/messages/ — сообщения
POST   /api/conversations/{id}/messages/ — отправить сообщение
PATCH  /api/conversations/{id}           — обновить статус
GET    /api/knowledge/                   — база знаний
POST   /api/knowledge/                   — добавить
POST   /api/knowledge/train              — обучить из диалога
GET    /api/notes/{phone}                — заметки по телефону
POST   /api/notes/                       — создать заметку
```

## 6. Генерация System Prompt

Ключевая логика — из данных формы автоматически генерируется промпт для AI.

**Входные данные:** form_data из заявки (JSON)

**Выходной промпт включает:**
- Роль: "Ты — AI-ассистент отеля {название}"
- Локация и описание
- Категории номеров с ценами по сезонам
- Правила бронирования (предоплата, отмена)
- Время заезда/выезда
- Удобства и услуги
- Как добраться
- Способы оплаты и реквизиты
- Стиль общения
- Языки
- Антигаллюцинация: "Не выдумывай информацию. Если не знаешь — скажи что уточнишь у менеджера"
- Booking flow: "Для бронирования собери: даты, кол-во гостей, ФИО, телефон"
- Тег эскалации: "Если не можешь помочь — напиши [НУЖЕН_МЕНЕДЖЕР]"

**Референс:** текущий промпт Тон Азура (проверен на 80+ реальных диалогах).

## 7. Мини-чат превью

**Как работает:**
1. Клиент заполняет форму (шаги 1-4)
2. На шаге 5 фронтенд отправляет `form_data` + сообщение на `/api/preview-chat/`
3. Backend генерирует промпт из `form_data` через promptGenerator
4. Отправляет в OpenRouter (Claude 3.5 Haiku)
5. Возвращает ответ в чат на странице

**Важно:**
- Не создаёт бота, не сохраняет в БД
- Использует общий API-ключ OpenRouter (расходы на Asystem)
- Лимит: 20 тестовых сообщений на заявку (защита от абуза)
- История чата хранится в state фронтенда (не на сервере)

## 9. Стек

| Компонент | Технология |
|-----------|-----------|
| Backend (единый) | FastAPI, SQLAlchemy async, PostgreSQL |
| AI | OpenRouter → Claude 3.5 Haiku (дефолт) |
| Telegram | Webhook (не polling) для multi-tenant |

| Хостинг | Contabo VPS (выделенный для ботов) |
| Reverse proxy | Nginx + Certbot |
| Контейнеризация | Docker Compose (опционально) |
